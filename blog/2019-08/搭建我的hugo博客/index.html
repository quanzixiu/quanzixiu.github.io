搭建我的hugo博客
  • Aug 28, 2019
  • 5 minutes read

准备工作

在准备之初首先需要

  • GitHub账号
  • VS code编辑器

下载安装git

git的安装非常简单，详细可参考下面给出的教程

git安装教程

下载安装hugo

  1. 下载hugo程序压缩包：hugo下载下载和自己系统匹配的版本

  1. 解压到某个文件当中

例如：D:\ProgramFiles\hugo

  1. 添加hugo到系统环境变量PATH中

找到“系统环境变量”的设置位置

  • 在开始菜单的搜索栏搜索环境变量

  • 右键我的电脑->属性->高级系统设置->环境变量

  1. 添加系统环境变量，依此：点击环境变量，找到系统变量中的path，点击编辑，然后点击新建，在使用浏览按钮选中文件夹，即可使用hugo。（选中到hugo.exe所在的文件夹即可，不需要选中hugo.exe）

    例如：D:\ProgramFiles\hugo

  2. 检查安装是否成功，在cmd中输入以下指令：

hugou version

如果得到如下响应，则说明安装成功

Hugo Static Site Generator v0.55.6-A5D4C82D windows/amd64 BuildDate: 2019-05-18T07:57:00Z

创建并配置站点

创建站点

在D:\ProgramFiles\hugo目录下，空白处右键，在右键菜单栏中选择git bash，打开git命令行，在命令行中输入如下指令

hugo new site XXXX

该命令执行后，便会在你选择的文件夹下生成名为“XXXX”的文件夹，我们在此称这个文件夹为站点文件的“根目录”。

下载并配置站点主题

下载主题

你可以在Hugo的主题库中选择你喜欢的主题，这里以AllinOne为例

在根目录下使用CD命令来到themes文件夹下，并且在AllinOne下很容易找到clone命令如下：

cd themes
git clone https://github.com/orianna-zzo/AllinOne.git themes/AllinOne

#以下为注释说明，可以在clone的整句命令后加上你要clone到的文件夹名字，默认是hugo-tranquilpeak-theme

配置主题

将 根目录\themes\AllinOne\exampleSite路径下的config.toml文件赋值到根目录下，覆盖掉根目录下的config.toml文件。然后，我们在vscode中打开并对其作一定的修改就可以直接使用，具体可以修改的内容如下（以我自己的站点配置文件为例，不建议直接复制，我也不清楚格式会不会影响，我会尽可能注释我知道的配置项的含义）：

builddrafts                = false
languageCode               = "zn-Hans"
canonifyurls               = true
contentdir                 = "content"
layoutdir                  = "layouts"
publishdir                 = "public"
enableEmoji                = true

summaryLength              = 200

# 每一页显示的文章数量
Paginate                   = 3


theme                      = "AllinOne"

[permalinks]
  blog                     = "blog/:year-:month/:slug/"

  # highlighting

设置文章模板

为了更好的使用附加功能，比如封面和缩略图，我们提前修改一下模板。这样，每次使用新建一篇文档时候就省去很多麻烦事。 使用vs code打开themes/tranquilpeak/archetypes中的default.md复制替换至 root\archetypes的default.md

---
title: "{{ replace .TranslationBaseName "-" " " | title }}"
date: {{ .Date }}
draft : false
# tags: ["css", "blog"]
# series: ["Example"]
# categories: ["Sci"]
# img: "images/blog/2018-08/test5.jpg"
toc: true
# summary: "This is an example of adding an intro picture to the post. "

---

接下来在根目录下使用以下命令生成一篇文档吧：

hugo new post/XXXX.md

待本地启动后，可以继续修改这篇文档的内容，在需要的文件夹中加入需要的图片，然后调整相关的参数，看看图片的几个功能是否正常或者符合预期。

启动博客的本地预览

建议在配置文件中设置好主题，或者使用 –t指令指定主题，在站点的根目录下使用命令进行本地启动，本地启动的命令如下：

hugo server -D

如果执行顺利，会有如下提示：

在浏览器中输入图片红框中的地址进行预览效果，比如一般是localhost:1313。

把博客部署到GitHub上

建立GitHub仓库

注册Github账户，并且建立一个Repository。仓库名为 (用户名).github.io

生成静态文件

接下来是生成静态文件，这里要把配置文件config.toml中的baseUrl修改为自己的仓库地址，即https：//XXXXXXXX.github.io/ 或者在hugo命令中使用指令指定 –baseUrl=“https：//XXXXXXXX.github.io/” 　　如果在配文件中已经设置好baseUrl和themes的路径，以及输出文件夹，可以直接在站点根目录下采用以下命令：

hugo

　　成功后会按照设置好的路径生成一个public文件夹，接下来我们需要将public文件夹中的内容提交到Github的仓库中去。

设置全局用户名和邮箱

　　使用以下命令配置全局用户名和邮箱，用于识别身份(XXXX是占位符，换成自己的github名字和邮箱)：

git config --global user.name "xxxxx"
git config --global user.email "xxxxx@xx.com"

将本地仓库文件部署到GitHub

　　然后进入public文件中依此使用以下命令部署到Github上：

#进入public文件夹
cd public
#初始化
git init
#添加public中的所有文件
git add .
#提交描述，XXXX中是你这次提交的描述，可以随便写
git commit -m "XXXXXXXX"
#链接到远端，把XXXX替换成自己的内容，不要忘了还有后缀.git
git remote add origin https://github.com/XXXXX/XXXXX.github.io.git

git pull origin master
git push -u origin master

　　这个过程一般不会有什么问题发生，第一次提交应该会要求输入Gihub的账号和密码验证，我使用的时候是直接弹窗的形式出现的，验证过后，之后的push就不用账号密码了。

后续更新内容

　　后续再写文章的时候，直接在站点根目录下新建一篇文档就行了（如果像前边介绍的配置了生成模板的话会很方便使用的）：

hugo new post/XXXXX.md

　　如果对配置不做大的改动（例如：更换主题等），后续的push过程很简单的啦，只有下面几个了(根目录下使用命令)：

hugo
cd public
git add .
git commit -m "XXXXXXX"
git pull origin master
git push origin master
